# Vrijgaveproces

De documentatiesite zou nu deel moeten uitmaken van het releasegesprek, ook al wordt deze niet in het binaire bestand van de app geleverd.

## Laat verwachtingen los

Elke release moet het volgende verifiëren:

- installatie- en configuratiestappen komen nog steeds overeen met de werkelijkheid
- functiedocumentatie weerspiegelt voor de gebruiker zichtbare wijzigingen
- technische documenten weerspiegelen in de architectuur zichtbare veranderingen
- release-opmerkingen of changelog-items worden bijgewerkt

## Aanbevolen toevoegingen aan de releasechecklist

- bevestig dat de Docusaurus-site netjes wordt gebouwd
- bekijk verbroken links voordat u deze publiceert
- update de berichten op de startpagina wanneer een release het productverhaal verandert
- zorg ervoor dat de implementatie van GitHub Pages slaagt na het samenvoegen

## Toekomstige evolutie

Naarmate het project volwassener wordt, kan dit gedeelte worden uitgebreid met:

- getagde release-opmerkingen
- docs-versiebeheerstrategie
- richtlijnen voor afschrijving
- upgrade-opmerkingen tussen app-releases
